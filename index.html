AISafe – Airspace Safety and Flight Operations System

Academic software engineering project

AISafe is designed to support the safe planning, validation and simulation of flight operations within controlled airspaces. The system models the aviation domain in a structured way, ensuring that flight operations are valid, safe and aligned with real-world aviation constraints.

Technologies

Java · Domain-Driven Design · EAPLI Principles · UML · PlantUML · Visual Paradigm · Markdown · Git

Technical Focus

  • Aircraft and fleet management
  • Flight routes, flights and flight plans
  • Air control areas, airports and weather scope
  • Flight plan validation and airspace simulation
  • Operational decision support

Key Highlights

  • Domain-driven system focused on aviation safety and operational correctness
  • Main chain: FlightRoute → Flight → FlightPlan → Validation / Simulation
  • Strong aggregate boundaries and business invariants
  • FlightPlan modelled as an entity inside Flight, not as an aggregate root
  • Designed to support future changes without major redesign

Logistics On Rails – Railway Logistics System

Academic integrative project

Logistics On Rails is a proof-of-concept software solution for logistics operators in railway infrastructure. The system supports railway infrastructure management, rolling stock, freight operations, warehouse management, routing, scheduling and station traffic security.

Technologies

Java · PL/SQL · C · Assembly · Arduino · Oracle LiveSQL · UML · PlantUML · Visual Paradigm · Git

Technical Focus

  • Railway infrastructure management
  • Warehouse management and freight dispatching
  • Train scheduling and route planning
  • Hardware and sensor integration
  • Spatial indexing and railway network optimization

Key Highlights

  • Multi-language project combining Java, PL/SQL, C, Assembly and Arduino
  • Modular architecture with database, business logic, station management and hardware layers
  • Warehouse operations using FEFO/FIFO logic
  • Spatial algorithms for station search, proximity search and network analysis
  • Developed using SCRUM, iterative development and Test-Driven Development principles
